# C# + Entity Framework Core Skill Guide

## Project Layout

```
ServiceName/
├── ServiceName.csproj
├── Data/
│   ├── AppDbContext.cs         # DbContext with DbSet<T> properties
│   └── Configurations/         # IEntityTypeConfiguration<T> per entity
├── Migrations/                  # Auto-generated by EF tooling
├── Models/                      # Domain entities
└── Repositories/                # Repository pattern over DbContext
```

## DbContext

```csharp
// Data/AppDbContext.cs
using Microsoft.EntityFrameworkCore;

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

    public DbSet<User> Users => Set<User>();
    public DbSet<Post> Posts => Set<Post>();
    public DbSet<Tag> Tags => Set<Tag>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        // Apply all IEntityTypeConfiguration<T> in this assembly
        modelBuilder.ApplyConfigurationsFromAssembly(typeof(AppDbContext).Assembly);
    }
}
```

## Fluent API Configuration

```csharp
// Data/Configurations/UserConfiguration.cs
using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Metadata.Builders;

public class UserConfiguration : IEntityTypeConfiguration<User>
{
    public void Configure(EntityTypeBuilder<User> builder)
    {
        builder.ToTable("users");
        builder.HasKey(u => u.Id);

        builder.Property(u => u.Email)
            .IsRequired()
            .HasMaxLength(255);

        builder.HasIndex(u => u.Email).IsUnique();

        builder.HasMany(u => u.Posts)
            .WithOne(p => p.Author)
            .HasForeignKey(p => p.AuthorId)
            .OnDelete(DeleteBehavior.Cascade);

        builder.OwnsOne(u => u.Address, address =>
        {
            address.Property(a => a.Street).HasMaxLength(200);
            address.Property(a => a.City).HasMaxLength(100);
        });
    }
}
```

## Entity Models

```csharp
// Models/User.cs
public class User
{
    public int Id { get; private set; }
    public string Name { get; private set; } = string.Empty;
    public string Email { get; private set; } = string.Empty;
    public Address? Address { get; private set; }
    public DateTime CreatedAt { get; private set; } = DateTime.UtcNow;
    public ICollection<Post> Posts { get; private set; } = new List<Post>();

    private User() { }  // EF Core needs parameterless constructor

    public static User Create(string name, string email) => new()
    {
        Name = name,
        Email = email,
    };
}

// Owned entity (maps to same table)
public record Address(string Street, string City, string PostalCode);
```

## LINQ Queries

```csharp
// Async queries — always use async overloads
var users = await _db.Users
    .Where(u => u.IsActive)
    .OrderBy(u => u.Name)
    .Skip((page - 1) * pageSize)
    .Take(pageSize)
    .ToListAsync(ct);

// Eager loading with Include
var posts = await _db.Posts
    .Include(p => p.Author)
    .Include(p => p.Tags)
    .ThenInclude(t => t.Category)
    .Where(p => p.Published)
    .ToListAsync(ct);

// Projection to DTO — avoids loading entire entity
var dtos = await _db.Users
    .Where(u => u.IsActive)
    .Select(u => new UserResponse(u.Id, u.Name, u.Email, u.CreatedAt))
    .ToListAsync(ct);

// Single entity
var user = await _db.Users.FindAsync(new object[] { id }, ct);
var user = await _db.Users.FirstOrDefaultAsync(u => u.Id == id, ct);
```

## Mutations

```csharp
// Create
var user = User.Create(req.Name, req.Email);
_db.Users.Add(user);
await _db.SaveChangesAsync(ct);

// Update — load, modify, save
var user = await _db.Users.FindAsync(new object[] { id }, ct)
    ?? throw new NotFoundException($"User {id} not found");
user.Update(req.Name);              // domain method keeps entity encapsulated
await _db.SaveChangesAsync(ct);

// Partial update (avoid loading all fields)
await _db.Users
    .Where(u => u.Id == id)
    .ExecuteUpdateAsync(s => s.SetProperty(u => u.Name, req.Name), ct);

// Delete
await _db.Users
    .Where(u => u.Id == id)
    .ExecuteDeleteAsync(ct);
```

## Migrations

```bash
# Add migration
dotnet ef migrations add AddUserTable --project ServiceName

# Apply to database
dotnet ef database update

# Generate SQL script (for CI/CD)
dotnet ef migrations script --idempotent -o migrations.sql

# Revert last migration
dotnet ef migrations remove
```

## Value Converters

```csharp
// Store enum as string in DB
builder.Property(u => u.Status)
    .HasConversion<string>()
    .HasMaxLength(50);

// Custom value converter
builder.Property(u => u.Tags)
    .HasConversion(
        tags => string.Join(",", tags),
        raw => raw.Split(',', StringSplitOptions.RemoveEmptyEntries).ToList()
    );
```

## Error Handling

- Catch `DbUpdateConcurrencyException` for optimistic concurrency conflicts.
- Catch `DbUpdateException` (wraps database-level errors like unique constraint violations).
- Never expose raw EF exceptions to API consumers — map them to domain exceptions.

## Key Rules

- Always call `SaveChangesAsync(ct)` — never the synchronous `SaveChanges()`.
- Use `IEntityTypeConfiguration<T>` in separate files — never configure entities inline in `OnModelCreating`.
- Use `.Select()` projection for read-only queries — avoid loading entire entity graphs when only a few fields are needed.
- Use `ExecuteUpdateAsync`/`ExecuteDeleteAsync` for bulk operations — avoid loading entities just to delete/update them.
- Register `DbContext` as `Scoped` (default) — never `Singleton`.
- Apply migrations at startup with `db.Database.MigrateAsync()` only in development — use idempotent SQL scripts in production CI/CD.
